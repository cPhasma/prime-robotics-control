# Что изменено в этой версии

## 1. Исправлена ошибка PWM
В серверном регуляторе теперь учитывается инверсная логика вашей прошивки/драйвера:

- `PWM = 0` — максимальная скорость;
- `PWM = 255` — минимальная скорость / фактически стоп.

Поэтому стоп-команды и аварийная остановка отправляют `255`, а увеличение управляющего сигнала уменьшает PWM.

## 2. Регулятор больше не падает из-за кратковременной потери дальномера
Файл Arduino `cmd_duration.cpp` теперь при потере эха возвращает не ошибку, а JSON:

```json
{"valid": false, "duration_us": 0, "distance_cm": -1}
```

Серверный регулятор при такой ситуации:

- не останавливает поток алгоритма;
- пишет warning в терминал и на сайт;
- берёт последнее корректное расстояние;
- если плохих чтений несколько подряд, временно ограничивает скорость.

## 3. Добавлена поддержка двух машинок
Сервер теперь держит два подключения:

- `car1`;
- `car2`.

Обе машинки подключаются к одному TCP-порту `5001` и одному сайту `http://localhost:5000`.

В файле ESP8266 перед прошивкой нужно выставить ID машинки:

```cpp
const char* carId = "car1";
```

Для второй машинки поменять на:

```cpp
const char* carId = "car2";
```

## 4. Добавлен параллельный запуск двух регуляторов
На сайте появилась кнопка:

```text
▶▶ Старт двух регуляторов
```

Она запускает два отдельных потока:

- регулятор для `car1`;
- регулятор для `car2`.

Они работают параллельно, а не по очереди.

## 5. FollowController вынесен из server.py
Теперь код регуляторов лежит отдельно:

```text
controllers/follow_base.py
controllers/follow_car1.py
controllers/follow_car2.py
```

Менять алгоритм первой машинки нужно в:

```text
controllers/follow_car1.py
```

Менять алгоритм второй машинки нужно в:

```text
controllers/follow_car2.py
```

`server.py` теперь отвечает в основном за сайт, TCP-подключения и маршрутизацию команд.

## 6. Графики и логи
Для каждой машинки на сайте отдельные графики:

- расстояние;
- состояния;
- управление;
- PWM.

В терминале логи идут с префиксом:

```text
[CAR1][FOLLOW]
[CAR2][FOLLOW]
```

Так проще понять, какая машинка что делает.

## 7. Прошивки ESP8266

В архиве есть два варианта ESP-моста:

```text
arduino/esp8266_bridge/esp8266_bridge/esp8266_bridge.ino              // car1
arduino/esp8266_bridge_car2/esp8266_bridge_car2/esp8266_bridge_car2.ino // car2
```

Первый прошивай в ESP первой машинки, второй — в ESP второй машинки.
