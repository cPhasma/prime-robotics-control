# WiFi Car Control System — guide по внесённым изменениям

## Что сделано

### Сервер
- Добавлен корректный TCP line-buffer: входящие JSON больше не режутся ошибочно по `recv()`.
- Добавлено ожидание ответа по `command_id`: для сценариев каждая команда теперь может выполняться по схеме `send -> wait result`.
- Добавлен Script Runner: запуск, пауза, продолжение, остановка пользовательских Python-скриптов.
- Добавлены сохранение, загрузка и удаление скриптов `.py`.
- Разрешены безопасные импорты из whitelist и пользовательские функции `def ...`.
- При ошибке скрипта или при ручной остановке сервер шлёт аварийный stop на моторы.

### ESP8266 bridge
- Убраны debug-строки в `Serial`, которые раньше засоряли канал между ESP и Arduino.
- Вместо `readStringUntil('
')` сделан неблокирующий буфер по символам.
- Серверный TCP поток и serial-поток теперь собираются построчно.

### Arduino Uno
- Убрано блокирующее чтение `Serial.readStringUntil('
')`, добавлен символьный буфер.
- Ответы `command_result` теперь экранируют JSON-строки `stdout/stderr`.
- `motor` возвращает короткий ответ `ok`, чтобы не перегружать канал при частых командах.
- `duration` должен возвращать JSON вида `{"duration_us":1234,"distance_cm":21.5}`.

## Структура проекта
- `server.py` — Flask + SocketIO + TCP сервер для ESP
- `templates/index.html` — веб-интерфейс
- `scripts/` — сохраняемые Python-сценарии
- `arduino/arduino_uno/...` — прошивка Arduino Uno
- `arduino/esp8266_bridge/...` — прошивка ESP8266

## Как это должно работать
1. Arduino поднимается и отправляет `capabilities`.
2. ESP соединяется по Wi‑Fi с сервером на TCP порту `5001`.
3. `server.py` принимает ESP, открывает веб-интерфейс на `http://localhost:5000`.
4. Из браузера ты можешь:
   - отправлять одиночные команды
   - сохранять Python-скрипты
   - запускать их, ставить на паузу, продолжать и останавливать
5. Скрипт вызывает `motor()`, `duration()`, `mpu6050()`, `ping()` как обычные Python-функции.
6. Сервер шлёт JSON-команду на Arduino и ждёт ответ именно с тем же `command_id`.

## Как запустить

### 1. Python
Установи зависимости:
```bash
pip install flask flask-socketio
```
При использовании NumPy/SciPy-сценариев дополнительно:
```bash
pip install numpy scipy
```

Запуск:
```bash
cd python_server_main_2
python server.py
```

### 2. Arduino Uno
Залей скетч из:
- `arduino/arduino_uno/arduino_uno/arduino_uno.ino`

Убедись, что рядом в проекте подключены:
- `commands.h`
- `cmd_motor.cpp`

- `cmd_mpu6050.cpp.cpp`
- `cmp_ping.cpp`

Перед прошивкой проверь:

- `arduino/esp8266_bridge/esp8266_bridge/esp8266_bridge.ino`

## Что проверить первым делом
1. Открыть веб на `http://localhost:5000`.
2. Убедиться, что ESP подключилась.
3. Нажать `Запросить команды`.
4. Убедиться, что появились `ping`, `motor`, `duration`, `mpu6050`.
5. Запустить короткий тестовый скрипт из `scripts/lqr_drive.py`.
6. Потом пробовать `scripts/follow_distance_demo.py`.

## Почему раньше терялись команды
- сервер резал TCP-поток по `recv()` как будто это отдельные готовые пакеты;
- ESP писала debug в тот же `Serial`, что использовался как транспорт к Arduino;
- на ESP и Arduino использовались блокирующие `readStringUntil`, из-за чего канал подвисал;
- `motor` посылал слишком длинные ответы для частого цикла управления.

## Что ещё может потребоваться руками
- В `cmd_mpu6050.cpp.cpp` и `cmp_ping.cpp` стоит привести имена файлов в порядок, чтобы не было путаницы.
- Если захочешь долгие сценарии, лучше перевести код Arduino с `String` на статические `char`-буферы.
- Для реального движения в колонне надо откалибровать HC-SR04 и bias гироскопа MPU6050.

## Важное замечание про stop
Остановка моторов в текущей логике — это `left_pwm=0, right_pwm=0`.
Не `255`.

## Новый плавный регулятор
Добавлен новый файл:
- `scripts/follow_distance_smooth.py`

Что в нём улучшено по сравнению с базовым `follow_distance_demo.py`:
- EMA-фильтрация дистанции HC-SR04
- EMA-фильтрация `gyro_z`
- калибровка bias гироскопа перед стартом
- deadzone по дистанции, чтобы машинка не дёргалась около цели
- ограничение производной ошибки дистанции
- ограничение скорости изменения средней тяги и поворотной добавки
- ограничение скорости изменения PWM по каждому борту
- мягкое удержание около уставки
- аварийный выход в stop при серии невалидных измерений расстояния

### Как запускать
Сначала лучше запускать именно:
```python
# в UI выбери этот файл
follow_distance_smooth.py
```

### Что должно измениться в поведении
- меньше рывков на разгоне и торможении
- меньше раскачки около целевой дистанции
- меньше паразитного подруливания из-за шума MPU6050
- более предсказуемая остановка при плохих данных с HC-SR04

### С чего начинать настройку
Если машинка всё ещё резкая:
- уменьши `KP_DIST`
- уменьши `KD_DIST`
- уменьши `MAX_US_STEP`
- уменьши `MAX_PWM_STEP`

Если машинка слишком вялая:
- немного увеличь `KP_DIST`
- немного увеличь `MAX_US_STEP`

Если машинка виляет:
- уменьши `K_THETA`
- увеличь `TURN_DEADBAND`
- уменьши `MAX_UD_STEP`

Если HC-SR04 шумит:
- увеличь `DIST_ALPHA` в сторону 0.45...0.55
- увеличь `DIST_DEADBAND_CM`
- уменьшай `KD_DIST`, если видны колебания
