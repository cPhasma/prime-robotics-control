# Исправление регулятора следования: MPU6050, задержка контура, переезд цели

## Что изменено

### 1. Гироскоп MPU6050

Файл:

```text
arduino/arduino_uno/arduino_uno/cmd_mpu6050.cpp.cpp
```

Изменения:

- Adafruit_MPU6050 заменён на I2Cdevlib:

```cpp
#include "I2Cdev.h"
#include "MPU6050.h"
```

- Добавлена однократная калибровка гироскопа:

```cpp
mpu.CalibrateGyro(6);
```

Она выполняется только при первом обращении к MPU6050 после запуска Arduino. Во время первого обращения машинка должна стоять неподвижно.

- Команда `mpu6050` теперь возвращает не сырые LSB, а угловые скорости в `rad/s`:

```json
{
  "gyro_x": ...,
  "gyro_y": ...,
  "gyro_z": ...,
  "omega_x": ...,
  "omega_y": ...,
  "omega_z": ...
}
```

### 2. Быстрый совместный опрос датчиков

Добавлена команда:

```text
sensors
```

Она за один запрос читает:

- HC-SR04;
- MPU6050.

Раньше регулятор делал два последовательных запроса: `duration`, потом `mpu6050`. Теперь основной цикл использует один запрос `sensors`, поэтому задержка контура меньше.

### 3. HC-SR04 больше не роняет алгоритм

Файл:

```text
arduino/arduino_uno/arduino_uno/cmd_duration.cpp
```

Если дальномер на мгновение теряет сигнал, команда возвращает:

```json
{"valid":false,"distance_cm":-1}
```

Но `exitCode = 0`, то есть это считается не аварией, а плохим измерением. Сервер использует предыдущее корректное расстояние.

### 4. Оценка рыскания

На сервере добавлены параметры:

- `gyro_axis`: `auto`, `x`, `y`, `z`;
- `gyro_sign`: `+1` или `-1`;
- `gyro_model_weight`.

Почему так: у MPU6050 акселерометр не может напрямую исправлять угол рыскания вокруг вертикальной оси. Акселерометр хорошо помогает для крена/тангажа, но не даёт абсолютный yaw. Поэтому для нашей задачи выбран практичный вариант:

```text
omega = gyro_model_weight * omega_gyro + (1 - gyro_model_weight) * omega_model
```

где:

```text
omega_model = (vR - vL) / W
```

То есть основной источник — гироскоп, а модель поворота используется как дополнительная стабилизация.

### 5. Борьба с переездом 1 метра

В продольный канал добавлены:

- прогноз расстояния вперёд:

```text
d_pred = d + d_dot * lookahead_s
```

- зона замедления:

```text
speed_limit = max_u * clamp((d_pred - d_target) / slow_zone_m, 0, 1)
```

- дополнительное торможение по скорости сближения:

```text
-k_brake * max(0, -d_dot)
```

- асимметричное ограничение PWM:
  - разгон медленнее;
  - торможение намного быстрее.

Это нужно потому, что в схеме PWM инверсный:

```text
PWM = 0   -> максимум
PWM = 255 -> стоп
```

Поэтому при приближении к цели сервер быстрее поднимает PWM к 255.

## Важные настройки для теста

Начать с таких значений:

```text
dt = 0.05
max_u = 0.45
slow_zone_m = 0.60
lookahead_s = 0.35
k_brake = 0.85
gyro_axis = auto
gyro_sign = +1
gyro_model_weight = 0.85
```

Если при ручном повороте машинки `theta` почти не меняется, значит выбрана не та ось. Тогда в интерфейсе по очереди проверить `z`, `x`, `y`.

Если машинку возвращает в сторону сильнее, а не выравнивает, поменять `gyro_sign` с `+1` на `-1`.

## Библиотеки Arduino

Для новой прошивки нужны библиотеки I2Cdevlib:

```text
I2Cdev
MPU6050
```

Их нужно установить в Arduino IDE в папку `libraries`.
