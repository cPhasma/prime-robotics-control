# Правка: матричная модель объекта и матричный закон управления

## Что изменено

Основной регулятор теперь приведён к виду:

```text
x(k+1) = Ad*x(k) + Bd*u(k) + Gd*vf(k) + Hd*omega_gyro(k)
y(k)   = C*x(k)
q(k)   = F*xi(k)
u(k)   = S*q(k)
PWM(k) = 255*(1-u(k))
```

где:

```text
x  = [vL, vR, theta, z, d]^T
u  = [uL, uR]^T
y  = [d, omega_model]^T
q  = [u_v, u_w]^T
xi = [vL, vR, theta, z, d, vf_hat, d_target]^T
```

## Файлы

```text
controllers/follow_base.py
```

Содержит матричную дискретную модель объекта:

```text
x(k+1)=Ad*x(k)+Bd*u(k)+Gd*vf(k)+Hd*omega_gyro(k)
y=C*x
```

Также там оставлена служебная часть: поток, чтение датчиков, логирование, отправка PWM, защита от пропажи дальномера/гироскопа.

```text
controllers/follow_car1.py
controllers/follow_car2.py
```

Содержат только матричный закон управления:

```text
q = F*xi
u = S*q
```

Для каждой машинки отдельный файл, чтобы можно было независимо менять коэффициенты.

## Датчики

Добавлена команда Arduino:

```text
sensors
```

Она за один запрос читает HC-SR04 и MPU6050. Это уменьшает задержку цикла управления. Если прошивка старая и команды `sensors` нет, сервер автоматически откатывается на старые команды `duration` + `mpu6050`.

MPU6050 работает через I2Cdevlib. Калибровка гироскопа выполняется один раз при первом обращении к датчику.

## PWM

Логика PWM сохранена инверсной:

```text
PWM = 0   -> максимум
PWM = 255 -> стоп
```

В коде используется нормированное управление:

```text
u = 0 -> стоп
u = 1 -> максимум
PWM = 255*(1-u)
```
