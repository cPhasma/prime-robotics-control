# Follow Controller Gyro Patch v4

Актуальная логика после патча MPU6050:

1. Калибровка гироскопа перенесена в прошивку Arduino.
   Используется I2Cdevlib: `mpu.CalibrateGyro(6)`.

2. Калибровка запускается один раз при первом вызове команды `mpu6050` после включения Arduino.
   При следующих замерах калибровка не повторяется.

3. Сервер больше не считает `gyro_bias` по серии измерений.
   Команда `mpu6050` сразу возвращает физическую угловую скорость в `rad/s`.

4. Для регулятора используются поля `omega_z` или `gyro_z`. Оба значения уже в `rad/s`.

5. Перед стартом регулятор останавливает моторы и делает первый вызов `mpu6050`, чтобы Arduino успела выполнить однократную калибровку.
   В этот момент машинка должна стоять неподвижно.

6. Timeout команды `mpu6050` увеличен до 8 секунд, потому что первый вызов может занять время из-за калибровки.

# Follow Controller Gyro Patch v3

Добавлено:

1. Калибровка нуля гироскопа перед стартом Follow Controller.
   - Моторы принудительно останавливаются.
   - Машинку нужно держать неподвижно.
   - Сервер собирает `gyro_z`, считает среднее значение и дальше вычитает его как `gyro_bias`.

2. Настройка количества сэмплов калибровки из интерфейса:
   - поле `Сэмплы калибровки gyro`, по умолчанию `60`.

3. Вывод в логи вектора состояния примерно раз в секунду:

   ```text
   [FOLLOW][x] vL=..., vR=..., theta=..., z=..., d=...; uL=..., uR=...; pwmL=..., pwmR=...; gyroRaw=..., gyroBias=..., omega=...
   ```

4. В telemetry добавлены поля:
   - `gyro_z_raw`
   - `gyro_bias`
   - `state_vector = [v_l, v_r, theta, z, d]`

5. В интерфейс добавлен KPI `gyro bias`.

Перед тестом: поставь машинку неподвижно, нажми старт Follow и не трогай её до завершения калибровки.
