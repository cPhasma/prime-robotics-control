# Патч MPU6050 через I2Cdevlib

Что изменено:

1. Файл `arduino/arduino_uno/arduino_uno/cmd_mpu6050.cpp.cpp` переписан с `Adafruit_MPU6050` на `I2Cdev.h` + `MPU6050.h`.

2. Калибровка гироскопа выполняется на Arduino через функцию библиотеки:

```cpp
mpu.CalibrateGyro(6);
```

Она запускается только один раз после включения Arduino — при первом вызове команды `mpu6050`. Дальше при каждом измерении калибровка уже не повторяется.

3. Во время первой калибровки машинка должна стоять неподвижно. Первый ответ `mpu6050` может идти дольше обычного, поэтому на сервере timeout для этой команды увеличен до 8 секунд.

4. Команда `mpu6050` теперь возвращает не сырые значения LSB, а угловую скорость в `rad/s`:

```json
{
  "valid": true,
  "unit": "rad/s",
  "calibrated": true,
  "calibrated_now": false,
  "gyro_x": 0.000000,
  "gyro_y": 0.000000,
  "gyro_z": 0.000000,
  "omega_x": 0.000000,
  "omega_y": 0.000000,
  "omega_z": 0.000000
}
```

Основное поле для регулятора — `omega_z` или совместимое старое поле `gyro_z`. Оба в `rad/s`.

5. Серверная калибровка `gyro_bias` отключена, потому что смещение теперь убирается на уровне прошивки. `FollowController` перед стартом только делает первый вызов `mpu6050`, чтобы прогреть датчик и запустить однократную калибровку.

6. Нужно установить библиотеки Arduino:

- `I2Cdev`
- `MPU6050` из I2Cdevlib / Jeff Rowberg

Старая библиотека `Adafruit_MPU6050` для этого файла больше не нужна.
