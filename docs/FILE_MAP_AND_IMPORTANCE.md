# Назначение файлов и папок проекта

В этом файле кратко описано, за что отвечает каждая основная часть репозитория. Степень важности указана для понимания того, какие файлы относятся к ядру системы, а какие являются вспомогательными.

## Уровни важности

| Уровень | Значение |
|---|---|
| Критический | Без этого файла или папки основная система не работает. |
| Высокий | Важная часть архитектуры, модели, датчиков или отладки. |
| Средний | Вспомогательные файлы, документация, тесты, примеры. |
| Низкий | Архивные материалы и дополнительные заметки. |

## Основные файлы

| Файл / папка | Важность | Назначение |
|---|---:|---|
| `server.py` | Критический | Flask-сервер: маршруты сайта, подключение машинок, отправка команд, запуск регуляторов. |
| `controllers/follow_base.py` | Критический | Общая база регулятора: цикл управления, чтение датчиков, обновление состояния, логирование. |
| `controllers/follow_car1.py` | Критический | Закон управления для первой машинки. |
| `controllers/follow_car2.py` | Критический | Закон управления для второй машинки. |
| `templates/index.html` | Высокий | Веб-интерфейс управления машинками. |
| `arduino/` | Критический | Код Arduino/ESP для обмена с сервером, работы моторов и датчиков. |
| `scripts/` | Средний | Вспомогательные расчёты и тестовые примеры. |
| `requirements.txt` | Средний | Python-зависимости проекта. |
| `.github/workflows/python-check.yml` | Средний | Автоматическая проверка синтаксиса Python-кода. |
| `versions/` | Высокий | Сохранённые версии проекта по этапам разработки. |
| `archive/` | Средний | Исходные архивы и контрольные суммы. |
| `CHANGELOG.md` | Высокий | Журнал изменений по версиям. |
| `DOSSIER_SOUTHBOARD.md` | Средний | Описание личного участка работы. |
| `docs/VERSION_MAP.md` | Высокий | Связь архивов, версий и выполненных изменений. |
| `docs/MEMORY_ESTIMATION.md` | Высокий | Оценка использования памяти Arduino Uno. |
| `docs/CODE_COMMENTARY.md` | Высокий | Пояснение ключевых частей кода. |
| `docs/TECHNICAL_NOTES.md` | Средний | Общие технические заметки по архитектуре. |

## Что относится к ядру проекта

К ядру проекта относятся файлы:

```text
server.py
controllers/follow_base.py
controllers/follow_car1.py
controllers/follow_car2.py
templates/index.html
arduino/
```

Именно эти части обеспечивают запуск сайта, подключение машинок, получение измерений и отправку команд управления.

## Что относится к математической части

К математической части относятся:

```text
controllers/follow_base.py
controllers/follow_car1.py
controllers/follow_car2.py
docs/VERSION_MAP.md
docs/TECHNICAL_NOTES.md
```

В этих файлах описана и реализована логика:

```text
x(k+1) = Ad*x(k) + Bd*u(k) + Gd*vf(k)
q = F*xi
u = S*q
```

## Что относится к аппаратной части

К аппаратной части относятся:

```text
arduino/
docs/MEMORY_ESTIMATION.md
docs/TECHNICAL_NOTES.md
```

Здесь описываются особенности Arduino/ESP, датчиков, PWM и ограничения по памяти.
