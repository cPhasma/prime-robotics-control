# Технические заметки

## Общая схема работы

Проект состоит из трёх основных уровней:

1. Веб-интерфейс.
2. Сервер управления.
3. Arduino/ESP и датчики.

Веб-интерфейс используется для запуска команд и отображения состояния. Сервер связывает интерфейс с машинками, запускает регуляторы и отправляет команды. Arduino/ESP выполняет низкоуровневые действия: моторы, датчики, обмен по последовательному каналу.

## Машинки

В системе используются идентификаторы:

```text
car1
car2
```

Каждая машинка имеет собственное подключение и собственный регулятор. Если подключена только одна машинка, сервер работает только с ней.

## PWM

В проекте используется инверсная PWM-схема:

```text
PWM = 0   -> максимальная скорость
PWM = 255 -> стоп
```

Поэтому при переводе нормированного управления используется:

```text
PWM = 255 * (1 - u)
```

где `u` находится в диапазоне от `0` до `1`.

## Дальномер

Дальномер используется для измерения расстояния до объекта впереди. Возможны кратковременные некорректные измерения, поэтому контроллер должен отделять одиночные ошибки датчика от реальной аварийной ситуации.

## MPU6050

MPU6050 используется для получения угловой скорости. В версии с I2Cdev калибровка выполняется один раз при первом обращении. Во время калибровки машинка должна быть неподвижна.

## Модель состояния

В качестве состояния рассматриваются величины, связанные со скоростью бортов, углом, боковым отклонением и расстоянием.

Общий вид дискретной модели:

```text
x(k+1) = Ad*x(k) + Bd*u(k) + Gd*vf(k)
y(k)   = C*x(k)
```

## Закон управления

Управление формируется через матричную схему:

```text
q = F*xi
u = S*q
```

`q` содержит продольную и угловую составляющие управления. Матрица `S` переводит их в команды левого и правого борта.

## Служебная логика

В программе остаются условия и проверки для:

- запуска и остановки потоков;
- проверки подключения машинок;
- обработки ошибок датчиков;
- ограничения диапазона PWM;
- логирования;
- отправки команд.

Эти проверки относятся к инфраструктуре программы. Сам расчёт управляющего воздействия вынесен в матричный закон.
