# Технические заметки по итоговой версии

## Основные файлы

```text
server.py
```

Веб-сервер, WebSocket, TCP-связь с ESP, маршрутизация команд между `car1` и `car2`.

```text
controllers/follow_base.py
```

Базовая инфраструктура регулятора: поток, чтение датчиков, фильтрация измерений, логирование, графики, отправка PWM. Здесь же реализована матричная дискретная модель объекта.

```text
controllers/follow_car1.py
controllers/follow_car2.py
```

Отдельные законы управления для каждой машинки. Управление формируется матрично:

```text
q = F*xi
u = S*q
```

```text
arduino/arduino_uno/arduino_uno/cmd_mpu6050.cpp.cpp
```

Чтение MPU6050 через I2Cdevlib. Калибровка гироскопа выполняется один раз.

```text
arduino/arduino_uno/arduino_uno/cmd_sensors.cpp
```

Совместный быстрый опрос HC-SR04 и MPU6050.

## Что говорить про if в коде

В коде остались `if`, но они относятся к инфраструктуре:

- проверка подключения;
- обработка ошибок датчиков;
- запуск/остановка потока;
- логирование;
- fallback при старой прошивке.

Это не нарушает требование к математическому закону управления, потому что само формирование `u` делается через матрицы.
