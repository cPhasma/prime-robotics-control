# Карта версий проекта

Эта таблица нужна, чтобы преподавателю было понятно: проект развивался не одним финальным архивом, а последовательными исправлениями.

| Версия | Назначение | Исходный архив | Что изменено |
|---|---|---|---|
| `v01_two_cars_base` | Базовая версия управления двумя машинками | `site_two_cars_fixed (1).zip` | Регулятор вынесен из server.py в controllers, добавлена поддержка car1/car2, параллельный запуск, защита от кратковременной ошибки дальномера. Историческая версия: PWM описан как обычный Arduino PWM, потом исправлялся. |
| `v02_pwm_inverted_fix` | Исправление инверсной PWM-логики | `site_two_cars_pwm_inverted_fixed.zip` | Сервер приведён к фактической логике прошивки/драйвера: PWM=0 — максимум, PWM=255 — стоп. Исправлены stop_pwm, norm_to_pwm, аварийная остановка и подписи на сайте. |
| `v03_matrix_control_law` | Матричный закон управления | `site_two_cars_math_control_fixed.zip` | Закон управления в follow_car1.py/follow_car2.py переписан через матричное умножение: q=F*xi, u=S*q. Условия if/elif вынесены из самой функции формирования управления. |
| `v04_mpu6050_i2cdev` | MPU6050 через I2Cdev | `site_two_cars_mpu6050_i2cdev_fixed.zip` | MPU6050 переведён с Adafruit_MPU6050 на I2Cdevlib, добавлена однократная калибровка CalibrateGyro(6), команда возвращает угловую скорость в rad/s. |
| `v05_single_car_capabilities` | Работа с одной машинкой и capabilities | `site_single_car_capabilities_fixed.zip` | Сервер больше не падает, если подключена только одна машинка. Запрос get_capabilities отправляется только реально подключённым машинкам, добавлен резервный список команд. |
| `v06_matrix_model_full` | Итоговая матричная модель и общий опрос датчиков | `L2_follow_controller_matrix_model_full (1).zip` | Итоговая полная версия: x(k+1)=Ad*x+Bd*u+Gd*vf+Hd*omega_gyro, y=C*x, q=F*xi, u=S*q. Добавлена команда sensors для совместного опроса HC-SR04 и MPU6050. |
| `v07_experimental_sensor_filter` | Экспериментальная правка фильтрации датчиков | `L2_follow_controller_sensor_filter_fix (1).zip` | Экспериментальная ветка по задержке контура, фильтрации MPU6050 и переезду целевого расстояния. Сохранена как отдельный snapshot, потому что структура отличается: регулятор частично находится в server.py. |

## Какую версию считать итоговой

Итоговая рабочая версия лежит в корне репозитория и соответствует:

```text
v06_matrix_model_full
```

Папка `v07_experimental_sensor_filter` сохранена отдельно как экспериментальная ветка по фильтрации датчиков и задержке цикла. Её лучше показывать как исследовательскую правку, а не как основной финальный код, потому что структура там отличается от основной архитектуры с отдельными `controllers/follow_car1.py` и `controllers/follow_car2.py`.

## Логика развития

1. Сначала была решена архитектура сайта и двух машинок.
2. Затем исправлена фактическая инверсная PWM-логика.
3. Потом закон управления был переписан в матричном виде.
4. После этого MPU6050 перевели на I2Cdev с однократной калибровкой.
5. Затем исправили работу с одной подключённой машинкой и список команд.
6. Финально модель объекта и закон управления оформлены как матричная реализация.
