# Карта версий проекта

В таблице указано, какой архив или папка соответствует каждому этапу разработки.

| Версия | Исходный архив | Основное содержание |
|---|---|---|
| `v01_two_cars_base` | `site_two_cars_fixed (1).zip` | Базовая версия для двух машинок, разделение регуляторов, параллельный запуск. |
| `v02_pwm_inverted_fix` | `site_two_cars_pwm_inverted_fixed.zip` | Исправление инверсной логики PWM. |
| `v03_matrix_control_law` | `site_two_cars_math_control_fixed.zip` | Матричный закон управления для `follow_car1.py` и `follow_car2.py`. |
| `v04_mpu6050_i2cdev` | `site_two_cars_mpu6050_i2cdev_fixed.zip` | Переход MPU6050 на I2Cdev, однократная калибровка, возврат `rad/s`. |
| `v05_single_car_capabilities` | `site_single_car_capabilities_fixed.zip` | Поддержка одной подключённой машинки и запрос списка команд. |
| `v06_matrix_model_full` | `L2_follow_controller_matrix_model_full (1).zip` | Итоговая версия с матричной моделью объекта и командой `sensors`. |
| `v07_experimental_sensor_filter` | `L2_follow_controller_sensor_filter_fix (1).zip` | Экспериментальная версия по фильтрации датчиков и уменьшению переезда цели. |

## v01_two_cars_base

Цель версии — перейти от управления одной машинкой к структуре, где можно подключать две машинки и запускать отдельные регуляторы.

Основные изменения:

- добавлены `car1` и `car2`;
- добавлен параллельный запуск регуляторов;
- код следования вынесен из сервера;
- добавлена обработка временной ошибки дальномера.

## v02_pwm_inverted_fix

Цель версии — исправить соответствие между нормированным управлением и PWM.

Особенность прошивки:

```text
PWM = 0   -> максимальная скорость
PWM = 255 -> стоп
```

Поэтому формула перевода должна учитывать инверсию.

## v03_matrix_control_law

Цель версии — привести закон управления к матричному виду.

Используется схема:

```text
q = F*xi
u = S*q
```

Это делает расчёт управления более формальным и отделяет математический закон от служебной логики программы.

## v04_mpu6050_i2cdev

Цель версии — улучшить получение данных с MPU6050.

Основные изменения:

- используется библиотека I2Cdev;
- калибровка выполняется один раз;
- возвращается угловая скорость;
- первый timeout увеличен из-за калибровки.

## v05_single_car_capabilities

Цель версии — сделать систему пригодной для работы с одной машинкой.

Основные изменения:

- сервер не падает при отсутствии второй машинки;
- список команд запрашивается у реально подключённых ESP;
- добавлен резервный список команд.

## v06_matrix_model_full

Цель версии — связать код регулятора с матричной моделью объекта.

Основные изменения:

- добавлено обновление состояния через матрицы;
- добавлена модель измерений;
- добавлена команда `sensors`;
- сохранена поддержка одной и двух машинок.

## v07_experimental_sensor_filter

Цель версии — проверить идеи по улучшению измерений и уменьшению переезда целевого расстояния.

Эта версия сохранена отдельно, потому что она относится к экспериментальной проверке и может отличаться от основной структуры проекта.
