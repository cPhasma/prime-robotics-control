# Пояснение ключевых частей кода

## `server.py`

`server.py` — центральная часть серверной программы. Он отвечает за:

- запуск Flask-приложения;
- отображение веб-интерфейса;
- хранение подключений машинок;
- отправку команд на ESP/Arduino;
- запуск и остановку регуляторов;
- получение списка доступных команд;
- работу с одной или двумя машинками.

Сервер не должен содержать весь математический закон управления. Его задача — организовать обмен данными и запустить нужный контроллер.

## `controllers/follow_base.py`

`follow_base.py` содержит общую логику регулятора следования. В нём находятся части, которые одинаковы для обеих машинок:

- цикл управления;
- чтение расстояния;
- чтение угловой скорости;
- проверка корректности измерений;
- обновление состояния;
- отправка PWM-команд;
- построение логов и графиков.

В итоговой версии состояние обновляется в матричной форме:

```text
x(k+1) = Ad*x(k) + Bd*u(k) + Gd*vf(k)
```

Это отделяет модель объекта от остальной серверной логики.

## `controllers/follow_car1.py` и `controllers/follow_car2.py`

Эти файлы содержат закон управления для каждой машинки. Отдельные файлы нужны, чтобы можно было независимо менять параметры регуляторов.

Общий вид закона:

```text
q = F*xi
u = S*q
```

где:

- `xi` — расширенный вектор состояния;
- `F` — матрица коэффициентов регулятора;
- `S` — матрица перехода от продольного и углового управления к левому и правому борту;
- `u` — нормированное управление бортами.

После расчёта управление ограничивается диапазоном и переводится в PWM.

## Перевод управления в PWM

В проекте используется инверсная логика PWM:

```text
PWM = 0   -> максимальная скорость
PWM = 255 -> стоп
```

Поэтому нормированное управление переводится так:

```text
PWM = 255 * (1 - u)
```

Если `u = 1`, получается `PWM = 0`, то есть максимальная скорость. Если `u = 0`, получается `PWM = 255`, то есть остановка.

## Обработка ошибок датчика расстояния

Дальномер может кратковременно вернуть некорректное значение. Такая ошибка не должна останавливать весь алгоритм. Поэтому служебная логика контроллера может использовать последнее корректное измерение или безопасное значение.

Это относится не к закону управления, а к инфраструктуре получения данных. Сам закон управления остаётся матричным.

## MPU6050

Для MPU6050 подготовлена версия с библиотекой I2Cdev. Основная логика:

- инициализация датчика;
- калибровка гироскопа один раз при первом обращении;
- получение угловой скорости;
- возврат значения в `rad/s`.

Калибровка не должна выполняться при каждом измерении, потому что это увеличивает задержку цикла управления.

## Команда `sensors`

Команда `sensors` нужна для объединённого получения данных:

```text
расстояние HC-SR04 + угловая скорость MPU6050
```

Это уменьшает число последовательных запросов и делает цикл управления более компактным.

## Разделение ответственности

В проекте используется такое разделение:

| Часть | Ответственность |
|---|---|
| Flask-сервер | Маршруты, подключения, отправка команд. |
| Контроллеры | Расчёт управления и цикл следования. |
| Arduino/ESP | Датчики, моторы, низкоуровневые команды. |
| Документация | Описание версий, модели и технических решений. |

Такое разделение упрощает отладку: если проблема в датчике, проверяется Arduino; если проблема в законе управления, проверяются `follow_car1.py` и `follow_car2.py`; если проблема в соединении, проверяется `server.py`.
