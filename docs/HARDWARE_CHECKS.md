# Вспомогательные проверки аппаратной части

В этой части проекта собраны небольшие утилиты для проверки поведения системы без запуска полного регулятора.

## Проверка PWM

В проекте используется инверсная логика PWM:

```text
PWM = 0   -> максимальная скорость
PWM = 255 -> остановка
```

Поэтому перед настройкой регулятора важно отдельно проверить преобразование нормированной команды управления `u` в значение PWM.

Файл:

```text
tools/krpff_checks/pwm_mapping_check.py
```

Он проверяет, что:

```text
u = 0.0 -> PWM = 255
u = 0.5 -> PWM примерно 128
u = 1.0 -> PWM = 0
```

## Проверка логов датчиков

При работе дальномер может временно не вернуть корректное значение. Для регулятора это опасно, потому что одиночный сбой не должен останавливать весь алгоритм.

Файл:

```text
tools/krpff_checks/sensor_log_analyzer.py
```

Он анализирует CSV-лог и считает:

- количество строк;
- количество пропусков дальномера;
- количество пропусков гироскопа;
- средний период опроса;
- максимальный разрыв между измерениями;
- участки, где датчик временно не давал данные.

## Быстрый запуск

Из корня репозитория:

```bash
python tools/krpff_checks/run_checks.py
```

Или отдельно:

```bash
python tools/krpff_checks/pwm_mapping_check.py
python tools/krpff_checks/sensor_log_analyzer.py tools/krpff_checks/sample_sensor_log.csv
```

## Практический смысл

Эти проверки не заменяют основной регулятор. Они нужны для отдельной проверки базовых вещей:

1. команда управления правильно переводится в PWM;
2. временные сбои датчиков видны в логах;
3. период опроса не становится слишком большим;
4. перед запуском регулятора можно быстро проверить входные данные.
