# Оценка использования памяти Arduino

Arduino Uno построена на ATmega328P. Для проекта важно учитывать ограниченный объём оперативной памяти, так как одновременно используются обработка команд, JSON-ответы, датчики и управление моторами.

## Аппаратные ограничения

Для Arduino Uno типовые ограничения такие:

| Тип памяти | Объём | Для чего используется |
|---|---:|---|
| Flash | 32 КБ | Код программы и строковые константы. |
| SRAM | 2 КБ | Глобальные переменные, стек, временные буферы. |
| EEPROM | 1 КБ | Долговременное хранение настроек. |

Самая критичная часть — SRAM. Если использовать слишком большие строки, JSON-документы или массивы, программа может работать нестабильно.

## Основные потребители SRAM

| Объект | Где используется | Оценка |
|---|---|---:|
| Буфер входящей команды | обработка команд ESP/Arduino | 128–256 байт |
| `StaticJsonDocument` | разбор и формирование JSON | 128–512 байт |
| Переменные датчиков | HC-SR04, MPU6050 | 20–80 байт |
| Переменные управления моторами | PWM, направление, режимы | 20–80 байт |
| Стек функций | вложенные вызовы | зависит от режима |

## Почему используются небольшие JSON-документы

В проекте нежелательно создавать большие JSON-буферы, потому что они занимают SRAM. Поэтому команды и ответы должны быть короткими.

Пример нормального ответа датчика:

```json
{
  "valid": true,
  "distance_m": 0.92,
  "omega_z": 0.01
}
```

Такой формат содержит только необходимые поля и не перегружает память.

## Использование `F()`

Строковые литералы в Arduino лучше помещать во Flash через макрос `F()`:

```cpp
Serial.println(F("MPU6050 calibration started"));
```

Это уменьшает расход SRAM, потому что строка не копируется в оперативную память.

## Использование `PROGMEM`

Если требуется хранить длинные постоянные таблицы или текстовые описания, их можно переносить во Flash через `PROGMEM`. В данном проекте основной упор сделан на короткие команды, поэтому большие таблицы в прошивке лучше не держать.

## Контроль свободной памяти

Для контроля SRAM можно использовать функцию `freeMemory()`. Она показывает примерный остаток свободной оперативной памяти во время работы программы.

Пример:

```cpp
int freeMemory() {
  extern int __heap_start, *__brkval;
  int v;
  return (int)&v - (__brkval == 0 ? (int)&__heap_start : (int)__brkval);
}
```

В коде это можно использовать при старте и после инициализации датчиков:

```cpp
Serial.print(F("Free memory: "));
Serial.println(freeMemory());
```

## Вывод по памяти

Основные меры, которые уменьшают риск переполнения SRAM:

- не хранить большие строки в оперативной памяти;
- использовать `F()` для постоянных сообщений;
- держать JSON-документы небольшими;
- не создавать крупные массивы в функциях датчиков;
- выполнять калибровку MPU6050 один раз, а не при каждом измерении;
- отдавать сложные расчёты серверу, а Arduino оставить работу с датчиками и моторами.

Такое разделение подходит для Arduino Uno: микроконтроллер выполняет низкоуровневые команды, а матричный регулятор работает на сервере, где ограничений по памяти значительно меньше.
